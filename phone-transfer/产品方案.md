# 📱➡️💻 局域网手机文件传输 Web 应用 — 产品方案

## 一、产品概述

一款基于局域网的轻量级 Web 文件传输工具。用户在电脑上启动服务后，手机扫描屏幕上的二维码即可打开上传页面，将图片和视频直接传输到电脑的图片文件夹中，无需数据线、无需安装 App、无需登录账号。

## 二、核心用户场景

| 场景 | 描述 |
|------|------|
| 日常传图 | 手机拍的照片想在电脑上编辑/备份 |
| 批量迁移 | 大量照片/视频一次性传到电脑 |
| 临时传输 | 借用他人电脑临时传几张图 |

## 四、功能清单

### 4.1 电脑端（服务端）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 一键启动 | `node server.js` 即可启动，零配置 | P0 |
| 二维码展示 | 终端内直接打印二维码，同时在浏览器控制页也展示 | P0 |
| 自动检测 IP | 自动获取局域网 IP，拼接访问地址 | P0 |
| 文件保存 | 默认存到 `~/Pictures/手机传输/`，按日期子目录归档 | P0 |
| 传输日志 | 实时显示文件名、大小、传输状态 | P1 |
| 自定义保存路径 | 启动时可通过参数指定保存目录 | P2 |
| 自动打开浏览器 | 启动后自动打开电脑端管理页面 | P2 |

### 4.2 手机端（上传页面）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 选择文件上传 | 支持从相册多选图片/视频 | P0 |
| 拍照直传 | 支持调用相机拍照后直接上传 | P0 |
| 上传进度 | 每个文件显示上传进度条 | P0 |
| 上传结果反馈 | 成功/失败状态提示 | P0 |

| 缩略图预览 | 上传前预览所选图片 | P1 |

## 五、技术选型

| 层面 | 技术 | 选型理由 |
|------|------|----------|
| 运行环境 | Node.js | 跨平台、生态丰富、安装普及率高 |
| HTTP 服务 | Express | 轻量、成熟、快速搭建 |
| 文件上传 | Multer | Express 标准文件上传中间件 |
| 二维码生成 | qrcode-terminal + qrcode | 终端打印 + 页面展示 |
| 前端页面 | 原生 HTML/CSS/JS | 零依赖，移动端兼容性最佳 |
| 网络发现 | Node `os.networkInterfaces()` | 自动获取局域网 IP |

## 六、交互流程

```
用户操作流程：

1. 电脑终端执行 → node server.js
                      │
2. 终端输出二维码 ←────┘
   + 显示访问地址 http://192.168.x.x:3000
                      │
3. 手机扫描二维码 ────►  打开上传页面
                      │
4. 点击「选择文件」 ───►  打开系统相册/文件选择器
                      │
5. 选中图片/视频 ────►  显示预览 + 文件列表
                      │
6. 点击「开始上传」 ───►  逐个上传，显示进度条
                      │
7. 上传完成 ─────────►  手机提示「✓ 全部上传成功」
                      │
8. 电脑终端输出 ←────── 「已接收 photo_001.jpg (3.2MB)」
                      │
9. 文件保存至 ────────► ~/Pictures/手机传输/2026-03-30/
```

## 七、文件存储策略

```
~/Pictures/手机传输/
├── 2026-03-30/
│   ├── IMG_0001.jpg
│   ├── IMG_0002.heic
│   └── VID_0003.mp4
├── 2026-03-31/
│   └── photo_001.png
└── ...
```

- **按日期归档**：自动创建日期子目录，便于管理
- **保留原文件名**：保持手机端原始文件名
- **重名处理**：同名文件自动追加序号，如 `IMG_0001(1).jpg`
- **格式支持**：jpg / jpeg / png / heic / gif / webp / mp4 / mov / avi / mkv

## 八、安全设计

| 风险 | 应对措施 |
|------|----------|
| 外网暴露 | 仅监听局域网 IP，不暴露公网 |
| 恶意文件 | 限制上传文件类型（仅图片/视频）|
| 文件过大 | 单文件默认限制 500MB，可配置 |
| 路径穿越 | 文件名净化，禁止 `../` 等路径字符 |
| 并发上传 | 限制并发连接数，防止资源耗尽 |

## 十、使用方式

```bash
# 1. 安装依赖
npm install

# 2. 启动服务
node server.js

# 3. 可选：指定保存目录和端口
node server.js --port 8080 --dir ~/Desktop/photos
```
